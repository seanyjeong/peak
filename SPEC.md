# MAXT (맥스트레이닝) - 제품 기획서 v1.0

> 체대입시 전문 트레이닝 관리 시스템
> 작성일: 2025-12-15

---

## 1. 프로젝트 개요

### 1.1 비전
"데이터 기반 체대입시 트레이닝의 새로운 기준"

FM2025가 축구 데이터를 축적하여 최적의 전술을 찾듯이,
MAXT는 체대입시 훈련 데이터를 축적하여 최적의 훈련 방법을 발견한다.

### 1.2 핵심 가치
- **간편함**: 트레이너가 현장에서 10초 만에 기록
- **축적**: 모든 훈련 데이터를 체계적으로 저장
- **발견**: 쌓인 데이터에서 성공 패턴을 찾아냄
- **연동**: P-ACA와 완벽한 데이터 동기화

### 1.3 목표
| 단기 (6개월) | 중기 (1년) | 장기 (2년+) |
|-------------|-----------|------------|
| MVP 출시 | 데이터 축적 | AI 추천 시스템 |
| 일일 훈련 기록 정착 | 패턴 분석 기능 | 개인 맞춤 프로그램 |
| P-ACA 연동 완료 | 성공 사례 DB 구축 | 타 학원 확장 가능 |

## 2. 사용자 분석

### 2.1 사용자 유형

#### 원장 (1명)
- **역할**: 전체 관리, 데이터 분석, 의사결정
- **주요 활동**: 통계 확인, 트레이너 관리, 학생 성장 모니터링
- **Pain Point**: 경험에만 의존하는 훈련 판단

#### 트레이너 (7명)
- **역할**: 일일 훈련 진행, 기록 입력
- **주요 활동**: 반 배치 확인, 훈련 계획 작성, 기록 측정
- **Pain Point**: 매번 수기 기록, 학생별 히스토리 파악 어려움

### 2.2 사용 환경
| 항목 | 내용 |
|------|------|
| 주 사용 기기 | 스마트폰, 태블릿 |
| 보조 기기 | PC (반 배치, 통계 확인) |
| 네트워크 | WiFi (안정적) |
| 사용 시간 | 수업 전 (배치), 수업 중/후 (기록) |

### 2.3 사용 시나리오

**시나리오 1: 수업 시작 전**
```
1. 원장/선임 트레이너가 오늘 출근 트레이너 체크
2. 각 트레이너가 오늘 훈련 계획 작성
3. 드래그앤드롭으로 학생 반 배치
4. 배치 완료 → 각 트레이너에게 알림
```

**시나리오 2: 수업 중**
```
1. 트레이너가 자기 학생 명단 확인
2. 훈련 진행
3. 기록 측정 시 바로 입력 (스마트폰)
4. 특이사항/컨디션 메모
```

**시나리오 3: 수업 후**
```
1. 오늘 훈련 내용 최종 저장
2. P-ACA 출결 자동 동기화
3. (선택) 학생별 메모 추가
```

```

3. JWT 토큰 발급
4. MAXT에서 토큰으로 인증
5. 일산맥스 소속 확인 후 접근 허용
```

### 6.3 출결 동기화
```
[MAXT에서 출석 체크 시]
1. MAXT daily_assignments 저장
2. P-ACA 출결 API 호출
3. P-ACA attendances 테이블 업데이트

[P-ACA에서 출석 체크 시]
1. P-ACA 출결 저장
2. Webhook 또는 폴링으로 MAXT 감지
3. MAXT daily_assignments 업데이트
```

## 12. 현실적인 데이터 수집 전략

### 12.1 핵심 문제

**문제**: 상세한 운동 기록이 필요하지만, 트레이너 입력 부담이 크면 더미 데이터만 쌓임

```
목표: "어떤 운동 → 어떤 결과" 상관관계 분석
현실: "상체 운동" 태그만 달면 → 쓸모없는 더미 데이터
필요: 구체적 운동 + 세트 + 횟수
```

### 12.2 해결책: 학원별 운동 라이브러리

**개념**: 학원이 자기만의 운동을 직접 등록하고, 트레이너는 선택만

```
학원 (원장) = 규격/뼈대
운동 라이브러리 = 그 규격에 맞게 생성
트레이너 = 라이브러리 가져다 사용
```

**등록 예시**:
```
A학원 운동 라이브러리:
├── 원장표 스쿼트 [하체][파워] - 3세트 10회 기본
├── 맥스 점프 [하체][순발력] - 박스 높이별
├── 특수 스트레칭 [유연성] - 15분 루틴
└── ...
```

**입력 플로우**:
| 단계 | 할 일 | 시간 |
|------|------|------|
| 최초 1회 | 학원 운동 등록 (원장) | 1~2시간 |
| 매일 | 운동 선택 + 세트/횟수 | 30초~1분 |

## 16. 운동 라이브러리 공유 시스템 (확장)

### 16.1 개념

```
플랫폼 = 운동 저장소
학원 = 프로젝트
운동 = 패키지

"인기 운동 팩" 제공 가능
```

### 16.2 공유 플로우

```
1. A학원이 "맥스 점프" 운동 등록
2. 효과 좋음 (사용 학생 기록 향상)
3. "공유" 버튼 → 전체 공개
4. B학원: "오 저거 좋아보이네" → 가져오기
5. B학원 라이브러리에 추가됨
```

### 16.3 추천 운동 팩

```
[체대입시 기본 팩] - 50개 운동
├── 하체 파워 (15개)
├── 상체 파워 (12개)
├── 민첩성 (10개)
├── 유연성 (8개)
└── 컨디셔닝 (5개)

→ 신규 학원 가입 시 기본 제공
→ 필요 없는 건 삭제, 추가 가능
```
